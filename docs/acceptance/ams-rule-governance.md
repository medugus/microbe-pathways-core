# AMS rule governance

## Purpose
This change adds **AMS rule governance** browser visibility so reviewers can understand which local stewardship rules exist, what they cover, and which recommendation each rule can generate.

## Browser-phase limitation
- This is browser-phase visibility for **local stewardship configuration**.
- Metadata and panel views are informative only.
- Production editing requires backend audit and permissions.
- Disabled-rule enforcement is metadata only pending engine enforcement.

## Rule metadata fields
Optional metadata now includes:
- `governanceStatus` (`active`, `review_only`, `draft`, `disabled`)
- `ruleCategory` (`restricted_approval`, `reserve_review`, `bug_drug_mismatch`, `de_escalation`, `syndrome_specific`, `reportability`, `safety_review`, `insufficient_data`)
- `ruleOwner` (`AMS`, `Microbiology`, `IPC`, `Joint`)
- `sourceLabel`, `reviewDate`, `lastReviewedBy`, `version`, `localPolicyRef`, `rationale`, `limitation`
- Scope fields for antibiotics/AWaRe/class, organisms/phenotypes, specimen/syndrome
- `recommendationCategory`, `approvalRequired`, `releaseReportImpact`

## How recommendations link to rules
- Stewardship recommendation output already carries `matchedRuleCode`.
- AMS recommendation cards now render:
  - `Generated by AMS rule: [RULE_CODE]`
- Cards link to matching governance cards in the panel (`#ams-rule-[RULE_CODE]`) when present.
- Mapping logic is done in `amsRuleGovernance` helper (not duplicated in React).

## Safety limits
- No AST interpretation or breakpoint logic changed.
- No prescribing behaviour change.
- If rationale/source is missing, UI shows:
  - `rationale not configured`
  - `source not configured`
- No server/auth/package-level changes.

## Manual validation checklist
1. Restricted meropenem recommendation shows generated rule code.
2. Reserve antibiotic review shows rule category and approval requirement.
3. Bug-drug mismatch recommendation links to a rule if available.
4. De-escalation opportunity links to a rule if available.
5. No-AMS-action case still works.
6. Governance panel shows rules missing rationale/source.
7. Existing AMS approval workflow still works.
8. Dashboard still shows AMS queue items.
9. No AST, breakpoint, IPC, specimen, validation, release, report/export, auth or package logic changed.

## Out-of-scope production claims
- No production rule editor.
- No claims of approved hospital policy.
- No automatic prescribing.
- No dose optimisation claims.
